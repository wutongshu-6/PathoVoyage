三、核心 REST API（与前端行为对应）

以下为与「智能分析」页强相关的接口约定（路径与字段可按团队规范微调）。

方法路径说明
GET/api/v1/cases/{case_id}病例详情（名称、切片编号、状态）。对应原 case_config 中的业务字段。
GET/api/v1/cases/{case_id}/viewer-config阅片器配置：全景图 URL、初筛/终筛热力图 URL、Patch 列表与描述、高危 Patch 列表等。前端用于替换静态 case_config_*.json。
POST/api/v1/cases/{case_id}/tasks创建分析任务，Body：{ "stage": "coarse" | "fine" }。返回 task_id。
GET/api/v1/tasks/{task_id}任务状态：pending|running|paused|completed|failed、总进度 0–100、当前子步骤名称。
POST/api/v1/tasks/{task_id}/pause暂停（推理节点协作取消或标记）。
POST/api/v1/tasks/{task_id}/resume继续。
GET/api/v1/tasks/{task_id}/events分页拉取已产生的步骤事件（日志行、热力图就绪、Patch 高亮指令等）。也可用 SSE：/api/v1/tasks/{task_id}/stream 推送。
GET/api/v1/cases/{case_id}/assets/{asset_key}或对资源返回 302 至带过期时间的签名 URL（热力图、Patch 图）。
POST/api/v1/cases/{case_id}/reports/generate触发生成诊断报告 PDF（异步）；完成后 reports 表更新。
GET/api/v1/reports/{report_id}/download下载诊断报告 PDF（权限校验）。对应「下载报告」按钮。
GET/api/v1/cases/{case_id}/documents/description下载病例描述 Word。对应「下载描述文件」。

四、核心后端逻辑说明（文字）

4.1 分析任务流水线

用户在前端点击「开始粗筛」或「开始细筛」后，API 在 analysis_tasks 中创建记录，状态为 running，并向队列投递作业。编排服务按阶段调用推理节点：有效组织提取 → 全局特征 → 可疑区域定位 → 热力值计算 → 图层对齐；粗筛输出初筛热力图与候选 Patch 列表，细筛对候选 Patch 做高分辨率推理与小热力图生成。进度与日志通过事件表或流式通道推送给前端，前端用进度条与「AI 诊断思考日志」区域展示，替代演示环境中的纯本地 JSON 时间线。

4.2 静态资源与缓存

热力图与 Patch 文件存放在对象存储；API 返回短期有效的签名 URL，避免浏览器缓存旧图（与前端 withNoCache 策略一致）。终筛热力图路径在配置中明确指向「最终概览」资源。

4.3 报告生成

「生成报告」触发生成任务：汇总病例元数据、各 Patch AI 结论、可选医生复核字段，套入病理报告模板生成 PDF，写入存储并在 reports 表记录版本。前端加载页展示前可先轮询「生成中」状态（与当前 10 秒过渡动画可对应真实异步）。

五、数据库设计（PostgreSQL 示例）

以下为与上述功能直接相关的核心表；字段类型为示意，实施时可按 DBA 规范调整。

5.1 用户与权限

-- 用户表
CREATE TABLE users (
  id              BIGSERIAL PRIMARY KEY,
  username        VARCHAR(64) NOT NULL UNIQUE,
  password_hash   VARCHAR(255) NOT NULL,
  display_name    VARCHAR(128),
  role            VARCHAR(32) NOT NULL DEFAULT 'pathologist', -- pathologist | admin
  is_active       BOOLEAN NOT NULL DEFAULT TRUE,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

5.2 病例与切片文件

CREATE TABLE cases (
  id              BIGSERIAL PRIMARY KEY,
  case_code       VARCHAR(64) NOT NULL UNIQUE,  -- 对应前端 case_id / 业务病例号
  case_name       VARCHAR(256) NOT NULL,
  slide_id        VARCHAR(128),
  cancer_type     VARCHAR(64),
  status          VARCHAR(32) NOT NULL DEFAULT 'draft',
  created_by      BIGINT REFERENCES users(id),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE wsi_files (
  id              BIGSERIAL PRIMARY KEY,
  case_id         BIGINT NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  storage_uri     TEXT NOT NULL,
  pyramid_meta    JSONB,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

5.3 分析任务与事件（对应粗筛/细筛、日志、进度）

CREATE TABLE analysis_tasks (
  id              BIGSERIAL PRIMARY KEY,
  case_id         BIGINT NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  stage           VARCHAR(16) NOT NULL,  -- coarse | fine
  status          VARCHAR(16) NOT NULL, -- pending | running | paused | completed | failed
  progress        NUMERIC(5,2) NOT NULL DEFAULT 0,
  current_step    VARCHAR(128),
  error_message   TEXT,
  started_at      TIMESTAMPTZ,
  finished_at     TIMESTAMPTZ
);

CREATE TABLE analysis_events (
  id              BIGSERIAL PRIMARY KEY,
  task_id         BIGINT NOT NULL REFERENCES analysis_tasks(id) ON DELETE CASCADE,
  seq             INT NOT NULL,
  event_type      VARCHAR(32) NOT NULL, -- log | step_progress | heatmap_ready | patch_highlight | ...
  payload         JSONB NOT NULL,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_analysis_events_task_seq ON analysis_events(task_id, seq);

5.4 分析产物元数据（热力图、Patch）

CREATE TABLE analysis_assets (
  id              BIGSERIAL PRIMARY KEY,
  case_id         BIGINT NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  task_id         BIGINT REFERENCES analysis_tasks(id),
  asset_type      VARCHAR(32) NOT NULL, -- stitch | heatmap_initial | heatmap_final | patch | patch_heatmap
  storage_uri     TEXT NOT NULL,
  extra           JSONB,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE patch_findings (
  id              BIGSERIAL PRIMARY KEY,
  case_id         BIGINT NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  patch_key       VARCHAR(256) NOT NULL,
  label           VARCHAR(256),
  description     TEXT,
  risk_level      VARCHAR(32),
  UNIQUE (case_id, patch_key)
);

5.5 报告与文档

CREATE TABLE reports (
  id              BIGSERIAL PRIMARY KEY,
  case_id         BIGINT NOT NULL REFERENCES cases(id) ON DELETE CASCADE,
  version         INT NOT NULL DEFAULT 1,
  status          VARCHAR(32) NOT NULL DEFAULT 'draft',
  pdf_storage_uri TEXT,
  diagnosis_summary TEXT,
  created_by      BIGINT REFERENCES users(id),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

六、核心后端代码示例（Python / FastAPI 风格，说明用）

下列为「创建分析任务」与「查询任务状态」的精简示例，用于文档或答辩时说明后端代码形态，非仓库内现成文件。

from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.orm import Session

router = APIRouter(prefix="/api/v1", tags=["tasks"])

@router.post("/cases/{case_id}/tasks")
def create_task(case_id: str, body: dict, db: Session = Depends(get_db), user=Depends(require_user)):
    case = get_case_by_code(db, case_id)
    if not case:
        raise HTTPException(404, "case not found")
    task = AnalysisTask(case_id=case.id, stage=body.get("stage", "coarse"), status="pending")
    db.add(task)
    db.commit()
    enqueue_analysis(task.id)  # 投递 Celery / RQ
    return {"task_id": task.id}

@router.get("/tasks/{task_id}")
def get_task(task_id: int, db: Session = Depends(get_db), user=Depends(require_user)):
    task = db.query(AnalysisTask).get(task_id)
    if not task:
        raise HTTPException(404)
    return {
        "status": task.status,
        "progress": float(task.progress),
        "current_step": task.current_step,
    }

七、前端与后端对接要点（实现层面）

  • 鉴权：所有 /api/v1 请求携带 Authorization: Bearer <token>。
  • analysis.html 中：将 loadCaseConfig 改为请求 /cases/{id}/viewer-config；将 executeScript 驱动的演示流改为「订阅任务事件流 + 根据事件类型更新 UI」，保留相同组件（进度条、日志、热力图、Patch 弹窗）。
  • 资源 URL：响应中返回 HTTPS 完整路径或签名 URL，前端继续用 <img> / 叠加层加载。
